# Teaching Visual Review Rubrics

这是一版给 `Manim4Teach` 使用的教学图评审规则初稿。

当前主要覆盖范围：

- 数学：解析几何、三角形与一般平面几何、函数图形、立体几何
- 物理：滑块、小球/小车、轨道运动、粒子运动、杆系/连杆运动

这里写的是“主要覆盖范围”，不是封闭列表；后续遇到相邻类型题目，可优先复用同一套规则，再按需要做局部扩展。

设计目标：

1. 先做小而硬的评审框架，不做庞大的学科知识库。
2. 区分公共规则与学科补充规则：
   - 公共规则：只保留跨数学/物理都高频、且具体可执行的少量问题
   - 数学规则：强调图-式对应、关系表达、推导承接
   - 物理规则：强调主体对象、状态变化、约束关系
3. 当前用途优先是：
   - 给后续 review prompt 提供固定 rubric
   - 给 revise prompt 提供稳定问题类别
4. 当前不直接等同于代码静态检查规则；这些条目首先服务于 review prompt 与 revise prompt。

推荐使用方式：

1. 先根据题目或 analysis_packet 判断领域：`math` / `physics`
2. 先评公共规则，再评领域规则
3. 最多输出 3-5 条高收益问题，避免修稿信号过载
4. 输出结构统一为：

```json
{
  "severity": "high|medium|low",
  "code": "focus_clear",
  "message": "当前画面主焦点不明确，多个对象同时竞争注意力。建议突出单一主体并弱化次要元素。"
}
```

注意：

- 这些规则评的是“教学图表达质量”，不是严格的数学正确性或物理正确性裁判。
- 如果题目本身是纯静态概念介绍，可放宽对“状态变化/运动过程”的要求。
