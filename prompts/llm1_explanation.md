# LLM1：explanation.txt

输出一个 `explanation.txt`，包含以下**固定标签**的段落。
要求：简洁、正确、结构化，便于后续抽取。

【硬性要求】

1) 只输出纯文本，不要 Markdown 标题/列表符号的装饰格式。
2) 必须包含全部标签，且标签顺序固定。
3) 每个标签下只写对应内容，不要重复题目原文。
4) 不要在文本中输出 JSON 或代码块。

[KNOWN]

逐条列出所有已知条件（含数值、变量、约束）。

[GOAL]

题目要求是什么（目标/求解对象）。

[PLAN]

解题高层大纲（3-7 条，短句即可）。

[DIAGRAM]

如果需要画图：画什么、哪些标注/参数关键；不需要则写“无”。

[STEPS]

可映射为 scenes 的编号步骤（1、2、3…），每步一句话。
