# 参考案例库（补充）

以下内容仅作补充参考，若与 `system.md` 有冲突，一律以 `system.md` 为准。

## 解题流程参考（默认）

以下流程作为解题教学视频的默认参考顺序，遇到题目较短或结构特殊时可局部合并：

1. `problem_intake`：完整呈现题意、已知和求解目标。
2. `goal_lock`：锁定当前小问目标量，明确本幕要解什么。
3. `model`：建立对象关系、变量符号与约束条件。
4. `method_choice`：说明采用哪条解法路线及其理由。
5. `derive`：分步推导，得到中间量与最终目标量。
6. `check`：做量纲/边界/代回等校验。
7. `recap`：回收本问结论，形成可复述答案。
8. `transfer`：提炼方法模板与易错点，迁移到同类题。

简化规则：

- 单问题可合并 `problem_intake + goal_lock`。
- 题目较短可合并 `check + recap`。
- 无论如何，`derive` 不可省略。

## 1) 布局案例（给 LLM 选型，不是硬模板）

### A. 左图右推导（题目讲解首选）

- 左侧主图区：约 55%-65% 宽度
- 右侧推导区：约 35%-45% 宽度
- 标题置顶，参数块放右上，结论放右下

### B. 上图下结论（概念可视化）

- 上半区：动态示意图
- 下半区：关键结论与简短公式
- 结论区始终不超过 3 行，避免文字墙

### C. 双列对比（易混概念/方法）

- 左列：旧视角或错误直觉
- 右列：正确视角或标准做法
- 用同位高亮引导“左右一一对应”

## 2) 动画段落案例（用于增强解释性）

### A. 关系建立段

- 先出现对象，再出现关系箭头/标记，再出现简短公式
- 结束时给短停顿，方便观众消化

### B. 关系变化段

- 通过 Transform / ReplacementTransform 展示“同一对象状态变化”
- 避免直接删旧建新导致视觉跳变

### C. 结论收束段

- 先弱化背景元素，再突出结论对象
- 用包围框/颜色强化“最终结论落点”

## 3) 反模式清单（用于生成前自查）

- 中心堆叠：大量对象都在中心，互相遮挡
- 节奏失衡：长时间静止或连续快切
- 文字先行：画面没建立就先堆大量文本
- 动画无解释：只有进出场，没有关系变化
- 主图抖动：频繁重建导致视觉不连续

## 4) 组合建议（启发式）

- 题目讲解：`A(左图右推导)` + `关系建立段` + `关系变化段` + `结论收束段`
- 概念讲解：`B(上图下结论)` + `关系建立段` + `结论收束段`
- 对比讲解：`C(双列对比)` + `关系变化段` + `结论收束段`
