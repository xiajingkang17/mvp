# Scene Type Templates

这些模板只用于提供布局参考，不是唯一标准，也不要求原样输出成字段。

你应该参考它们来形成一段更贴近当前 scene 的 `layout_prompt`，再输出实际 `layout_contract`。

所有模板都默认继承同一条硬规则：

- 底部 `subtitle` zone 必须固定为 `x0=0.05, x1=0.95, y0=0.02, y1=0.12`
- 任何模板都不得改动这个字幕区

## 1. `problem_split_left_right`

适用于整题开场。

推荐布局：

- 左侧：题干全文
- 右侧：分问列表 / 每问任务
- 上方：标题可省略；如果已有完整题干，不要再额外加大标题
- 主图：若需要，仅作为辅助区，不要压到题干上
- 若某一步需要局部放大主图或切入问题卡，用 `step_layouts` 做局部调整，不要重画整套 zone

## 2. `question_entry_top_left`

适用于某一问刚开始。

推荐布局：

- 左上角：当前小题题干卡
- 主区：题图 / 坐标系 / 轨迹
- 右侧或下方：目标量提示
- 若后续 step 要转入推导，只在 `step_layouts` 中把公式组切入独立区，不要破坏左上角题干卡

## 3. `diagram_formula_split`

适用于“主图 + 推导”并行的讲解幕。

推荐布局：

- 一侧：主图
- 另一侧：公式区
- 公式区纵向排布
- 主图与公式区必须明显分离
- 不同 step 可通过 `step_layouts` 调整公式区占比，但不要让公式重新压回主图区

## 4. `full_formula_stack`

适用于纯推导幕。

推荐布局：

- 主要区域给公式区
- 公式按纵向堆叠
- 当前问题卡可保留在上角小区域

## 5. `check_summary_split`

适用于检查与总结。

推荐布局：

- 左侧：检查或验证
- 右侧：本问结论或答案汇总

## 6. `diagram_only_focus`

适用于纯图示、纯轨迹、纯结构说明。

推荐布局：

- 主图区居中
- 标签沿对象周边摆放
- 不要让说明文字占据主图中心
