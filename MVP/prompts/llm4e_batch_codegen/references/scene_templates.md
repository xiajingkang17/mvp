# 常见 scene 模板

这些模板只用于提供 scene 代码生成时的画面组织参考，不是唯一标准，也不要原样输出成字段。  
你应该参考它们来组织当前 scene 的对象分布、step 推进和视觉重心，再根据 `layout_contract` 落成代码。

## 1. 第一幕：题干与问题总览

适用于整题开场。

推荐组织：

1. 优先题干与问题总览，不要额外加大标题去抢空间
2. 左侧放完整题干或已知条件
3. 右侧放每一问的任务列表
4. 如需主图，只作为辅助区，不要压到题干上
5. 充分利用上方空间，不要把第一幕所有东西都堆到中间

## 2. 每问开场：题目卡 + 主图

适用于某一问刚开始讲解。

推荐组织：

1. 当前问题卡优先放左上角
2. 主图放中部或右侧
3. 目标量可以放在问题卡下方或公式区上方
4. 不要一上来先堆公式
5. 这一幕的任务是让观众先知道“这一问在解什么”

## 3. 图示 + 推导并行

适用于主图与推导同时存在的 scene。

推荐组织：

1. 主图在一侧
2. 公式区在另一侧
3. 公式区纵向排布
4. 主图与公式区必须明显分离
5. 当前 step 若切入更多公式，只扩大公式区，不要让公式重新压回主图

## 4. 纯推导幕

适用于以公式推导为主的 scene。

推荐组织：

1. 主要区域给公式堆栈
2. 当前问题卡可保留在左上角或上方小区域
3. 公式逐行出现，不要整组同时出现
4. 结论行单独高亮

## 5. 检查 / 总结幕

适用于结果检查、答案汇总、方法总结。

推荐组织：

1. 结论优先
2. 若仍保留主图，主图应弱化或缩小
3. 总结文字和最终公式不要互相压住
4. 检查区和结论区尽量分开

## 6. 纯图示 / 纯运动幕

适用于结构展示、轨迹展示、受力示意、运动预演。

推荐组织：

1. 主图优先，占据主要视觉空间
2. 标签沿对象边缘摆放
3. 说明文字少而精，不要占主图中心
4. 如果有轨迹或运动，必须让运动可见，不要只剩静态点和标签
