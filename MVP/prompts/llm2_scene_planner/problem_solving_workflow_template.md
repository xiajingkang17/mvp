# 题解教学视频工作流模板（供 LLM2 规划 scene）

这份模板不是让你机械照抄，而是规定“解题教学视频通常如何推进”的默认骨架。

你应该先沿着这个骨架规划 scene，再根据题型做压缩、合并或局部创新。

## 0. 总体认知顺序

一条标准题解教学视频，通常按下面的顺序推进：

1. 看清题：题目说了什么、问了什么
2. 形成直觉：这个过程大概怎么发生
3. 建模：对象、相互作用、约束、坐标系、守恒量
4. 求解：推导、计算、分段讨论
5. 验证：量纲、极限、方向、物理意义
6. 总结迁移：方法模板、易错点、变式

你在规划 scene 时，应优先保证这条认知链存在，而不是只追求画面炫技。

## 1. 开头：题目呈现 + 是否需要预演

### 1.1 必备：完整题目呈现

职责：

- 把题目完整、可读、无干扰地呈现出来
- 让观众知道有几问、最终要算哪些量
- 如果题图复杂，先建立阅读入口

对应 `workflow_step`：

- `problem_intake`

### 1.2 什么情况下优先 `preview_first`

满足任一条就优先考虑安排 `preview`：

- 过程强时序：多阶段运动、碰撞、反弹、分段受力、进出场区
- 轨迹不直观：抛体、圆周、带电粒子轨迹、相对运动
- 后续推导依赖全局直觉
- 题图复杂，先看过程能降低后续建模负担

`preview` 的职责：

- 只让观众先看懂“起点 -> 过程 -> 关键事件 -> 终点”
- 不承担正式推导
- 不在这里展开完整解题

### 1.3 什么情况下优先 `model_first`

更适合直接进入建模：

- 静态平衡、纯电路稳态、纯能量守恒一段到位
- 题目本身图文已经足够直观
- 必须先澄清对象与条件，否则动画会误导

## 2. 从“整题”进入“逐问”

多问题通常需要从整题视角切到单问视角。

推荐动作：

- 明确当前在解哪一问
- 说明这一问与前一问结果的关系
- 如果后续要复用中间量，要在这里埋下钩子

常用 `workflow_step`：

- `goal_lock`
- `recap`

## 3. 每一问的标准局部结构

对每一问，优先沿着下面的局部结构规划 scene。可以合并相邻阶段，但不要打乱角色。

### Q-0: 问题重述与目标锁定

职责：

- 用一句话说清楚这一问到底要求什么
- 从题干中筛出与当前问题直接相关的条件

对应 `workflow_step`：

- `goal_lock`

### Q-1: 建模与画图

职责：

- 选对象系统、受力/场、约束、参考系
- 把语言条件翻译成图上元素、几何关系与符号

对应 `workflow_step`：

- `model`

### Q-2: 策略选择

职责：

- 明确采用哪条主线：动力学、能量、动量、几何、轨迹方程等
- 给出一句“为什么这样做”的理由

对应 `workflow_step`：

- `method_choice`

### Q-3: 推导 / 计算

职责：

- 完成核心推导链和计算
- 每一步只引入一个新结论
- 关键中间量要有明确名字和物理意义

对应 `workflow_step`：

- `derive`

### Q-4: 检验与解释

职责：

- 做量纲、极限、方向、物理意义一致性检查
- 把结果翻译成一句可理解的话

对应 `workflow_step`：

- `check`

### Q-5: 小结

适用场景：

- 本问引入了可复用技巧
- 本问结果是下一问的地基
- 推导较长，需要收口

对应 `workflow_step`：

- `recap`

## 4. 结尾固定结构

结尾至少应覆盖：

1. 答案汇总
2. 方法模板
3. 易错点或变式

常用 `workflow_step`：

- `recap`
- `transfer`

## 5. 如何把工作流真正写进 scene

不要把工作流藏在脑子里，必须直接体现在每个 scene 的 JSON 字段里。

每个 scene 至少要显式给出：

- `workflow_step`: 这一幕属于哪一个教学步骤
- `question_scope`: 这一幕服务哪一问或哪一段整题任务
- `scene_goal`: 这一幕的教学目标
- `entry_requirement`: 进入这一幕前观众已经知道什么
- `scene_outputs`: 这一幕结束时产出哪些认知结果
- `handoff_to_next`: 下一幕如何承接
- `layout_prompt`: 这一幕的大体布局和信息组织方式
- `panels`: 这一幕必须出现哪些面板（panel 级，不是 object 级）
- `beat_sequence`: 这一幕内部先呈现什么、再切换什么、最后收在哪里

如果读者从 `stage2_scene_plan.json` 里看不出整题教学流程，就说明你的规划还不够具体。

## 6. 面板级 + 节拍级约束

你应该把每个 scene 规定到“面板级 + 节拍级”，但不要越界到 object 级。

### 面板级

你要明确：

- 这一幕有哪些信息面板
- 这些面板各自占据什么抽象区位
- 哪个面板承担题干、主图、推导、检查或总结

### 节拍级

你要明确：

- 这一幕先呈现什么
- 然后强调什么
- 哪些面板会出现、冻结、高亮、更新或退出
- 每个认知动作大致持续多久

### 明确禁止

- 不要规定 object id
- 不要规定具体动画函数
- 不要把 beat 写成代码执行步骤
