# 历史错误案例 003：文本/公式重叠导致可读性差

## 真实问题（已发生）

- 右侧连续 `to_edge(RIGHT).shift(DOWN*...)` 堆叠多个公式，发生重叠或越界。
- 标题、策略、总结同时保留在同一区域，遮挡主图。

## 根因

- 逐条硬编码位置，没有做对象宽高检查与已放置对象冲突检测。
- 没有按“主图区 / 公式区 / 结论区”分区管理。

## 生成硬约束（必须遵守）

1) 必须在代码中实现基于包围盒的防重叠放置逻辑（宽高/边界检查）。
2) 多行公式必须优先 `VGroup(...).arrange(DOWN, buff=...)` 后整体定位。
3) 同一时刻屏幕信息量受控：能分步展示就不要同屏堆满。
4) 如果对象仍拥挤，优先缩放或分批淡入，而不是继续叠加。

## 推荐实现要点

```python
def is_overlap(a, b, pad=0.12):
    return not (
        a.get_right()[0] + pad < b.get_left()[0]
        or a.get_left()[0] - pad > b.get_right()[0]
        or a.get_top()[1] + pad < b.get_bottom()[1]
        or a.get_bottom()[1] - pad > b.get_top()[1]
    )
```

## 输出前自检清单（必须执行）

- 右侧公式是否使用了 `VGroup(...).arrange(DOWN, ...)`？
- 是否存在明显重叠/越界的 `Text/MathTex`？
- 是否把互斥信息拆成多步展示？
