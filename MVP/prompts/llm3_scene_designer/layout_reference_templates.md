# 布局参考模板（仅参考，不是硬枚举）

以下模板是高优先级构图参考。你可以基于它们自由变化与混合，不要机械照抄。
但无论怎样变化，最终都要落实为带 `role` 的 `zones`。

## 模板 A：左图右公式（常规首选）

- 适用：主图讲解 + 右侧公式推导并存的常规场景。
- 推荐 zone role 组合：
  - `main`
  - `formula`
  - 可选 `title` / `summary`
  - 建议 `subtitle`
- 主图在左，公式推导在右。
- 左侧主图宽度建议占画面 `55% ~ 60%`。
- 右侧用于公式推导、要点文字、结论。
- 避免主图横向铺满全屏，尤其不要让主图宽度接近屏幕总宽度。

## 模板 B：长图下置（长轨道/长装置场景）

- 适用：主图天然是长条形、横向跨度很大时。
- 推荐 zone role 组合：
  - `main`
  - `title`
  - `formula` 或 `summary`
  - 建议 `subtitle`
- 主图放在下方区域，建议高度约占画面 `30%`。
- 上方区域用于标题、公式推导、关键结论。
- 目标是避免“长图压缩公式区”或“图文互相覆盖”。

## 模板 C：左上标题 + 左下图 + 右区推导

- 适用：讲解 + 推导并行的分镜。
- 推荐 zone role 组合：
  - `title`
  - `main`
  - `formula`
  - 可选 `summary`
  - 建议 `subtitle`
- 左上放标题或本段目标。
- 左下放主图（实验示意、轨迹图、装置图）。
- 右侧区域集中放公式推导与关键步骤。

## 模板 D：全屏过程动画（仅过程展示）

- 适用：整个屏幕几乎都用于展示动画过程，重点是运动、变形、轨迹、过程演示。
- 更适合放在开头用于“先给观众看完整动画效果”，或放在结尾用于“完整轨迹回放 / 收尾总结”。
- 推荐 zone role 组合：
  - `animation_only`
  - 可选很薄的 `title` 或 `aux`
  - 若需要字幕，额外拆出底部 `subtitle`
- 可以让主视觉占据绝大部分屏幕。
- 这种模板只适合“过程展示型” scene。
- 一旦同一 scene 需要稳定的公式推导、结论面板或长驻文字说明，就不要使用单一全屏 `animation_only` 模板，而应拆分出 `formula/title/summary` 等核心 zone。
- 不要把“中央大图/全屏主视觉”当成默认布局；它只在完整动画展示时成立。

## 使用原则

- 以上只作为构图参考，不限制创意发挥。
- 若 scene 的主要目标是推导、解释、分步分析，优先参考模板 A/B/C，并在模板基础上自由发挥。
- 若 scene 的主要目标是展示完整动画效果、开头总览或结尾收尾，才优先考虑模板 D。
- 在自由发挥时，优先保持可读性：主图、公式、标题三者不重叠。
- 若存在字幕/讲解文本，字幕区固定放在屏幕最下方，并作为保留区，不得与其它区域重叠。
- `zones[*].id` 可以自由命名，但 `zones[*].role` 必须明确表达分区职责。
- 若存在公式推导与主图并存，优先拆成至少两个核心 zone，而不是单一 `center/full_screen` zone。
- 若参考模板与当前 scene 叙事目标冲突，可在模板基础上调整比例与分区，但不要直接退化成中央大图默认布局。
